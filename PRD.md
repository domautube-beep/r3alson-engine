## 1. 제품 비전

> "Suno로 음악을 만드는 크리에이터가 **감성은 지키면서**, 데이터의 힘으로 돈 되는 곡을 만들 수 있도록 돕는다."

### 핵심 철학: 데이터는 나침반, 운전은 사용자

```
틀린 방식: "이거 만들어" (데이터가 결정)
맞는 방식: "이런 게 뜨고 있어. 네 스타일로 해볼래?" (데이터 + 자유도)

데이터가 방향을 제안하고, 사용자가 최종 결정한다.
앱은 조언자이지 지시자가 아니다.
```

### 시장 기회

2026년 3월 기준, Suno는 유료 구독자 200만 명, ARR $300M, 전년 대비 404% 성장 중이다.
이 시장에서 **"데이터 기반 프롬프트 생성기"** 카테고리의 직접 경쟁사는 현재 0개다.
HitCraft는 이 빈 카테고리의 첫 번째 표준 제품을 목표로 한다.

## 2. 문제 정의

### 누가 겪는 문제인가
Suno로 AI 음악을 만드는 크리에이터. 이 중 스트리밍 수익화를 목표로 하는 30만 명이 핵심 타겟이다.

### 어떤 상황에서
"이번에 어떤 곡을 만들까?" 를 결정할 때, 그리고 결정 후 프롬프트와 가사를 작성할 때.

### 무슨 불편을
- 뭘 만들어야 스트리밍이 나올지 모른다 (감으로 장르 선택)
- Suno 프롬프트 작성이 시행착오 투성이다
- sunoPrompt(스타일 태그)와 productionNote(제작 지시) 구분을 모른다
- 가사를 쓰는 데 시간이 오래 걸린다
- 제목/태그를 어떻게 써야 Spotify 알고리즘에 걸리는지 모른다

### 왜 지금 해결해야 하는가
- Suno 프롬프트 생성기 5개+ 존재하지만 전부 "그럴듯한" 프롬프트만 준다
- "지금 뭐가 뜨는지" 데이터 기반 프롬프트 생성기는 **현재 0개**
- 프롬프트 하나로 300만 스트림 달성 사례 다수 → 프롬프트 품질이 수익을 결정

## 4. 타겟 사용자 (3 페르소나)

### 페르소나 A: 수익화 추구형 크리에이터 "릴슨"
- **프로필**: 20~30대, Suno + DistroKid 사용, 월 50~200달러 스트리밍 수익 목표
- **사용 빈도**: 주 2~3회 (새 곡 제작 사이클)
- **핵심 니즈**: "이번 달 어떤 장르 만들어야 스트리밍 나와?"
- **불편 포인트**: 트렌드를 모른다, 프롬프트 반복 실패, 가사에 시간 낭비
- **앱 사용 시나리오**: 트렌드 확인 → 하이브리드 모드 → 가사 생성 → 프롬프트 복사 → Suno

### 페르소나 B: 다작형 아마추어 "민준"
- **프로필**: 30~40대, 퇴근 후 취미, Suno 월 구독, DistroKid 미사용
- **사용 빈도**: 월 4~8회 (주말 제작)
- **핵심 니즈**: "빠르게 완성된 느낌의 곡을 만들고 싶다"
- **불편 포인트**: 프롬프트 짜는 시간이 아까움, 가사는 영어라 부담
- **앱 사용 시나리오**: 트렌드 모드 → 주제 입력 → 한 번에 결과 패키지

### 페르소나 C: API 없는 탐색자 "지아"
- **프로필**: 20대, Claude API 키 없음, 처음 써보는 사용자
- **사용 빈도**: 비정기 (먼저 가치 확인 후 결제 결정)
- **핵심 니즈**: "API 키 없어도 핵심 기능을 느낄 수 있어야 한다"
- **불편 포인트**: 가입부터 막힘, 비용 불명확
- **앱 사용 시나리오**: 데모 모드 → 샘플 프롬프트 확인 → 가치 확인 → 30일 무료 체험 시작

### MVP v1.0 기능 목록

#### F1. 실시간 트렌드 대시보드

**사용자 스토리**
> "수익화를 목표로 하는 크리에이터로서 이번 주 Spotify에서 성장 중인 장르를 확인하면 어떤 장르로 곡을 만들지 데이터로 결정하고 싶다"

**상세 요구사항**
- 정상 흐름: 앱 진입 → Top 5 장르 성장률 표시 (예: dark ambient +18%) → 개인 스타일 매칭 추천 1개 → "새 곡 만들기" CTA
- 예외 흐름: Spotify API 오류 시 최근 캐시 데이터로 표시 + "X일 전 기준" 레이블
- 엣지 케이스: 데이터 없음 시 "트렌드 분석 중" 표시, 성장률 동률 시 절대 수치 높은 순 정렬

**수용 기준**
- [ ] Top 5 장르 + 성장률(%) 표시
- [ ] 데이터는 최대 7일 이내 기준
- [ ] 개인화 추천 1개 (아티스트 설정 기반)
- [ ] 성장률 하락 장르는 회색으로 표시 (빨간색 금지 — 자유 선택 존중)
- [ ] 트렌드 없이 열면 의미 없다는 UX 원칙 충족: 매일 열 이유가 있어야 함

**모바일 필수 체크**
- [ ] 375px에서 Top 5 장르 전부 스크롤 없이 확인 가능
- [ ] 터치로 장르 탭 → 상세 정보 슬라이드업
- [ ] 키보드 미사용 화면 (입력 없음)

#### F2. 3모드 선택 시스템

**사용자 스토리**
> "크리에이터로서 내가 원하는 만큼만 데이터에 의존하면 앱이 내 개성을 살리면서도 수익성을 높여주기를 원한다"

**상세 요구사항**
- 트렌드 모드: 장르/무드/BPM 전부 앱 자동 세팅. 사용자는 주제/감정만 입력
- 자유 모드: 사용자가 장르/무드 직접 선택. 앱은 Suno 최적화만 담당. 선택한 장르 성장률을 회색으로 참고 표시
- 하이브리드 모드 (기본값 추천): 앱이 뼈대 제안 → 사용자가 원하는 항목 수정
- 어떤 모드든 최종 결과물(프롬프트+가사+메타) 형식 동일
- 예외 흐름: 모드 변경 시 기존 입력값 초기화 여부 확인 팝업 ("지금까지 입력한 내용이 초기화됩니다. 계속하시겠어요?")

**수용 기준**
- [ ] 3가지 모드 선택 카드 (터치 타겟 80px 이상)
- [ ] 하이브리드 모드에 "추천" 뱃지
- [ ] 모드 변경 시 입력값 초기화 확인 팝업
- [ ] 자유 모드에서 하락 장르 선택 시 회색 정보 표시 (경고 X)

#### F3. Suno 최적화 프롬프트 생성

**사용자 스토리**
> "Suno 사용자로서 수노에서 바로 붙여넣기만 하면 좋은 결과가 나오는 프롬프트를 받으면 시행착오 없이 빠르게 곡을 만들고 싶다"

**상세 요구사항**
- 출력 형식: sunoPrompt(스타일 태그)와 productionNote(제작 지시) 분리 표시
- sunoPrompt 예시: `dark ambient, dreamy, melancholic, minor key, 75 BPM, atmospheric pads, distant reverb, cinematic, ethereal`
- productionNote 예시: `Intro에서 sparse piano로 시작, Chorus에서 full reverb pad 레이어 추가, 2:45 fade out`
- Suno Master Architect v3.0 엔진 적용: 7재료 공식 (장르+템포+무드+악기+보컬+시대+레퍼런스)
- 키워드 순서 최적화 (장르 > 템포/BPM > 무드 > 악기 > 보컬)
- 충돌 키워드 조합 자동 차단 (예: "lo-fi"+"128 BPM"+"aggressive")
- 금지 단어 이중 방어 적용
- 예외 흐름: 생성 실패 시 에러 메시지 + 재시도 버튼 (자동 재시도 1회)
- 엣지 케이스: 키워드 15개 초과 시 우선순위 낮은 것부터 자동 제거 + 알림

**수용 기준**
- [ ] sunoPrompt와 productionNote 분리 출력
- [ ] 각 필드 수정 가능 (인라인 편집)
- [ ] 1탭 복사 (sunoPrompt 복사 / productionNote 복사 / 전체 복사 각각)
- [ ] 생성 시간 3초 이내 (4G 기준)
- [ ] Suno v5 문법 기준 유효성 검증

**모바일 필수 체크**
- [ ] 375px에서 두 필드 구분 명확히 표시
- [ ] 복사 버튼 최소 44×44px
- [ ] 키보드 열릴 시 복사 버튼 가림 없음

#### F4. AI 가사 생성 (가사 엔진 v82.0)

**사용자 스토리**
> "크리에이터로서 주제나 감정을 입력하면 수익화 가능한 보편적 감정 기반 가사 초안을 받으면 내 스타일로 빠르게 다듬고 싶다"

**상세 요구사항**
- 입력: 주제/감정 자유 텍스트 (최대 200자)
- 가사 옵션 4종: 인스트루멘탈(가사 없음), AI 생성, 직접 입력, 내 가사 + AI 보완
- AI 생성 시 감정 아크 자동 적용 (12종 중 선택 — 기본값은 장르 기반 자동 매칭)
- 출력 포맷: [Verse 1], [Pre-Chorus], [Chorus], [Verse 2], [Bridge], [Outro] Suno 태그 자동 적용
- 한국어 입력 → 영어 가사 생성 (한국어 음절 규칙 엔진 내장)
- 랩 장르 선택 시 펀치라인 생성 모드 자동 활성화
- 줄 단위 수정/삭제/재생성 가능
- 보편적 인간 감정 기반 (특정 인물/지명 직접 참조 금지 — 스트리밍 보편성 확보)
- 예외 흐름: 금지 단어 포함 가사 생성 시 해당 줄 자동 대체 + "[편집됨]" 표시
- 엣지 케이스:
  - 입력 비어있을 때: 주제/감정 입력 유도 메시지 표시
  - 입력 200자 초과: 201자 이후 입력 차단 + 카운터 표시
  - 특수문자/이모지 입력: 허용 (Suno 호환 여부 체크 후 경고)

**수용 기준**
- [ ] [Verse][Chorus] 등 Suno 구조 태그 자동 포함
- [ ] 줄 단위 탭 → "수정 / 이 줄만 재생성 / 삭제" 메뉴
- [ ] "전체 재생성" 버튼
- [ ] 직접 입력 모드 전환 시 기존 AI 가사 보존 (비교 가능)
- [ ] 가사 전체 1탭 복사

**모바일 필수 체크**
- [ ] 375px에서 가사 에디터 정상 표시
- [ ] 줄 탭 액션 메뉴 터치 타겟 44×44px
- [ ] 키보드 열릴 때 현재 편집 줄이 가려지지 않음 (스크롤 자동 조정)
- [ ] 모바일 와이어프레임 참조: 섹션 6

#### F5. AI 메타데이터 생성 (Spotify SEO)

**사용자 스토리**
> "스트리밍 수익화를 목표로 하는 크리에이터로서 Spotify 알고리즘에 최적화된 제목/설명/태그를 받으면 검색 노출과 추천 확률을 높이고 싶다"

**상세 요구사항**

- Spotify 태그 자동 생성 (15개 이내, SEO 최적화)
- 예외 흐름: 모두 마음에 안 들 때 "다시 생성" 버튼

**수용 기준**
- [ ] 제목 후보 3개 라디오 선택
- [ ] 직접 입력 옵션 항상 표시
- [ ] 태그 개별 삭제 가능 (X 버튼)
- [ ] 메타데이터 전체 1탭 복사

#### F6. 결과 패키지 + 다음 단계 가이드

**사용자 스토리**
> "처음 Suno + DistroKid를 사용하는 크리에이터로서 한 화면에서 필요한 것을 모두 복사하고 다음에 뭘 해야 하는지 보이면 Suno에서 바로 실행할 수 있다"

  3. "DistroKid에 업로드 (제목+태그 복사해서 사용)"

**수용 기준**
- [ ] 한 화면에 모든 복사 버튼 노출
- [ ] 가이드 3단계 이내
- [ ] 저장 버튼 → 히스토리 기록
