# PRD: R3ALSON Engine — AI 음악 히트메이커

---

## 1. 제품 비전
> "AI 음악 크리에이터가 **자기 감성은 지키면서**, 데이터의 힘으로 돈 되는 곡을 만들 수 있도록 돕는다."

### 핵심 철학: 데이터는 나침반, 운전은 사용자

```
❌ "이거 만들어" (로봇처럼 데이터만 따라감)
✅ "이런 게 뜨고 있어. 네 스타일로 해볼래?" (데이터 + 자유도)

데이터가 방향을 제안하고, 사용자가 최종 결정한다.
앱은 조언자이지 지시자가 아니다.
```

---

## 2. 문제 정의
- **누가**: 수노(Suno)로 AI 음악을 만드는 크리에이터 (100만+ 유료 사용자)
- **어떤 상황에서**: "이번에 어떤 곡을 만들까?" 고민할 때
- **무슨 불편을**:
  - 뭘 만들어야 스트리밍을 받을지 모름 (감으로 장르 선택)
  - 수노 프롬프트 작성이 시행착오 투성이
  - 가사를 쓰는 데 시간이 오래 걸림
  - 제목/태그를 어떻게 써야 Spotify 알고리즘에 잡히는지 모름
  - 프롬프트 생성기는 있지만, "돈 되는" 관점이 없음
- **왜 지금**:
  - 수노 프롬프트 생성기 5개+ 존재하지만 전부 "그럴듯한" 프롬프트만 줌
  - "지금 뭐가 뜨는지" 데이터 기반 프롬프트 생성기는 **제로**
  - 프롬프트 하나로 300만 스트림 달성 사례 → 프롬프트 품질이 수익을 결정

## 3. 성공 지표

### 북극성 지표
> "이 앱의 프롬프트로 만든 곡의 평균 스트리밍이 앱 없이 만든 곡보다 2배 이상 높다"

| 지표 | 목표 | 측정 방법 |
|------|------|----------|
| 앱 프롬프트 곡 vs 일반 곡 스트리밍 비교 | 2배+ | 배포 후 30일 스트리밍 추적 |
| 주간 앱 사용 | 매주 1회+ | 앱 접속 로그 |
| 프롬프트 → 수노 → 배포 전환율 | 70%+ | 프롬프트 생성 후 실제 배포까지 완료 비율 |
| 사용자 프롬프트 수정률 | 30~60% | 자유도 활용 지표 (0%면 로봇, 100%면 앱 무용) |
| 모바일 사용 비율 | 60%+ | analytics |

## 4. 사용자 여정 (User Journey)

### 시나리오: 릴슨이 이번 주 곡을 만든다

```
1. [앱 접속] "이번 주 뭘 만들까?"
   → 트렌드 대시보드:
     "🔥 dark ambient +18% | lo-fi jazz +12% | ethereal pop +9%"
     "💡 추천: dark ambient — 네 스타일(chill/melancholic)과 맞고, 지금 성장 중"
   → 느낌: "오 dark ambient가 뜨는구나"

2. [모드 선택 — 사용자 자유도의 핵심]

   ┌─────────────────────────────────────────┐
   │  어떻게 만들까요?                          │
   │                                           │
   │  🎯 트렌드 모드                            │
   │  "데이터가 추천하는 대로"                    │
   │  → 장르/무드/BPM 자동 세팅                  │
   │                                           │
   │  🎨 자유 모드                              │
   │  "내 감성대로, 데이터는 참고만"               │
   │  → 내가 장르/무드 선택, 앱은 최적화만         │
   │                                           │
   │  🔀 하이브리드 모드 (추천)                   │
   │  "데이터 제안 + 내 터치"                     │
   │  → 앱이 제안하면 내가 수정                    │
   │                                           │
   └─────────────────────────────────────────┘

3. [하이브리드 모드 선택]
   → 앱 제안:
     장르: Dark Ambient        [수정 가능]
     무드: Melancholic, Eerie  [수정 가능] → 릴슨이 "Dreamy" 추가
     BPM: 75                   [수정 가능]
     분위기: Cinematic          [수정 가능]
   → 느낌: "앱이 뼈대를 주고, 내가 살을 붙이는 느낌"

4. [가사 생성]
   → 옵션:
     ○ 가사 없음 (인스트루멘탈)
     ● AI 가사 생성 → 주제/감정 입력: "새벽 3시의 고독"
     ○ 내 가사 직접 입력
     ○ 내 가사 + AI 보완

   → AI 가사 생성 후:
     [Verse 1]
     empty streets below my window
     shadows dancing with the rain
     ...

     → 사용자가 한 줄씩 수정 가능
     → "이 줄 다시 생성" 버튼 (마음에 안 드는 줄만 재생성)
     → Suno [Verse][Chorus][Bridge] 태그 자동 포맷

5. [프롬프트 생성]
   → 앱이 최종 프롬프트 생성:
     "dark ambient, dreamy, melancholic, eerie, minor key,
      75 BPM, atmospheric pads, distant reverb,
      cinematic, ethereal, 2:45"
   → 수노 최적화 적용 (키워드 순서, 조합 최적화)
   → 프롬프트 수정 가능 (고급 사용자용)
   → [📋 프롬프트 복사] 1탭

6. [메타데이터 생성]
   → 제목 후보 3개 (SEO 최적화):
     ● "Hollow Echo"
     ○ "3AM Reverie"
     ○ "The Last Signal"
   → 또는 직접 입력
   → 설명 + 태그 자동 (Spotify 알고리즘 최적화)
   → 전부 수정 가능

7. [결과 패키지]
   → 한 화면에:
     ✅ 프롬프트 (복사 가능)
     ✅ 가사 (복사 가능)
     ✅ 제목 + 설명 + 태그 (복사 가능)
     📖 "수노에서 이 프롬프트로 곡 만들기" 가이드
     📖 "DistroKid에 올리기" 가이드

8. [배포 후 추적] (v1.5)
   → "이 프롬프트로 만든 곡" 태그
   → 30일 후 스트리밍 결과 연결
   → "트렌드 모드 곡이 자유 모드보다 평균 2.3배 스트리밍 높았습니다"
```

| ID | 기능 | 설명 | 자유도 | 수용 기준 |
|----|------|------|--------|----------|

| F3 | AI 프롬프트 생성 | 장르+무드+BPM → 수노 최적화 프롬프트 | 모든 파라미터 수정 가능 | - [ ] 7개 재료 구조 적용<br>- [ ] 수노에서 바로 사용 가능<br>- [ ] 1탭 복사 |
| F4 | AI 가사 생성 | 주제/감정 → 가사 → Suno 포맷 | 줄 단위 수정/재생성 가능 | - [ ] [Verse][Chorus] 자동 포맷<br>- [ ] 줄별 재생성 버튼<br>- [ ] 직접 입력 옵션 |
| F5 | AI 메타데이터 생성 | 제목 3종 + 설명 + 태그 (SEO 최적화) | 전부 수정/직접 입력 가능 | - [ ] Spotify SEO 키워드 포함<br>- [ ] 1탭 복사 |
| F6 | 결과 패키지 + 가이드 | 프롬프트+가사+메타 한 화면 + 수노/DistroKid 가이드 | — | - [ ] 한 화면에 모든 것<br>- [ ] 가이드 3단계 이내 |
| F7 | 프롬프트 히스토리 | 과거 만든 프롬프트 목록 + 메모 | — | - [ ] 히스토리 저장<br>- [ ] 검색 가능 |

### 자유도 설계 원칙

```
모든 AI 생성 결과에 적용되는 규칙:

1. 제안이지 강제가 아니다
   → 모든 필드에 "수정" 버튼
   → 모든 선택에 "직접 입력" 옵션

2. 데이터는 보여주되, 결정은 사용자가 한다
   → "dark ambient가 +18% 성장 중" (정보)
   → "dark ambient로 만드시겠어요?" (선택권)
   → "아니, 나는 synthwave로 할래" (존중)

3. 수정 비율을 추적한다
   → 수정 0%: 사용자가 로봇처럼 따르고 있음 (나쁜 신호)
   → 수정 30~60%: 이상적 (데이터 + 개성)
   → 수정 100%: 앱이 쓸모없음 (나쁜 신호)
   → 이 데이터로 AI 제안 품질을 개선

4. 자유 모드에서도 데이터를 보여준다
   → "synthwave를 선택하셨네요. 현재 Spotify 성장률: -3%"
   → 경고가 아닌 정보. 그래도 synthwave로 할 수 있음.
   → "트렌드와 다른 선택도 가치 있습니다. 차별화 전략이 될 수 있어요"

5. 가사는 특히 자유도가 높아야 한다
   → AI 가사 = 초안. 사용자가 살을 붙이는 재료
   → 줄 단위 수정/삭제/재생성
   → "이 줄만 다시" 버튼
   → 완전 직접 작성 옵션
```

| Trend Up | #10B981 (초록) | +% 성장 |
| Trend Down | #6B7280 (회색) | -% (빨강 아님 — 자유 모드 존중) |

### 트렌드 하락 표시 주의
```
❌ 빨간색으로 경고: "synthwave -3% ⚠️" → 사용자 위축
✅ 회색으로 정보: "synthwave -3%" → 참고만, 선택은 자유
   + "트렌드와 다른 선택도 차별화 전략이 될 수 있어요" 문구
```

## 8. 기술 설계

### 아키텍처
- **프레임워크**: Next.js 14+ (App Router)
- **스타일링**: Tailwind CSS
- **데이터**: Supabase (PostgreSQL)
- **인증**: Supabase Auth
- **호스팅**: Vercel (무료)
- **외부 API**:
  - Claude API — 프롬프트/가사/메타데이터 생성
  - Spotify Charts (크롤링 또는 3rd party) — 트렌드 데이터
  - DALL-E 3 — 커버 아트 (v1.5)

### 트렌드 데이터 수급 (MVP)
```
MVP 현실적 접근:
  1순위: 수동 업데이트 (주 1회, 공개 차트 데이터 기반)
  2순위: Spotify Web API (제한적이지만 무료)
  3순위: Chartmetric API (유료, $50/월~)
  4순위: 자체 크롤링 (리스크 있음)

MVP에서는 1순위(수동)로 시작. 검증 후 자동화.
```

### 수노 프롬프트 최적화 엔진 (핵심 IP)
```

1. 7재료 공식: [장르]+[템포]+[무드]+[악기]+[보컬]+[시대]+[레퍼런스]
2. 키워드 순서 최적화 (장르가 먼저, 무드가 뒤)

5. 장르별 최적 BPM/키/곡길이 매핑
6. v5 최신 프롬프트 문법 반영

이 엔진이 ChatGPT와의 핵심 차별점.
```
